# Security Policy

OpenClaw Frontier Stack is designed around sanitized examples and explicit release gates.

## Reporting

Report vulnerabilities through the repository security advisory flow when available. If advisories are not enabled, contact the maintainers through the public channel listed by the repository owner.

## Secret handling

Never open an issue or pull request containing live credentials, OAuth tokens, private keys, private hostnames/IPs, chat IDs, memory dumps, transcripts, raw logs, vector stores, backups, or customer data.

If a secret is accidentally exposed, remove it, rotate it outside this repository, and note only the class of issue in public discussion.
